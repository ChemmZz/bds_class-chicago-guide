01 — Concept

The brief: a city that resists simplification

Chicago is not one thing. It is the tallest skyscrapers and the oldest blues bars. It is architectural pilgrimages and neon-lit cocktail dens. It is Sunday morning Gospel and Saturday night techno in a basement below a rock venue.

Most tourist guides flatten this. They optimize for accessibility over honesty, for TripAdvisor ratings over texture. The brief here was different: build a tool that respects the city's complexity — that lets a visitor choose their own register and receive a coherent, optimized day in return.

"Give me the jazz and the doors — I'll take care of the rest."

The output should feel less like a listicle and more like a note passed to you by someone who actually lives there. Opinionated. Specific. With tips that only make sense if you've actually stood in front of the thing.

02 — Curation

25 spots across 6 categories — why these, why not others

The six tags — Architecture, Music, Nightlife, Jazz, Doors, Café — were chosen to cover different visitor modes and times of day, while remaining specific enough to produce useful filtering.

"Doors" is deliberately unconventional. Chicago's ornamental metalwork tradition — Sullivan, Burnham, Root — is one of the city's most overlooked pleasures. The category exists to surface spots that reward slowing down and looking closely. Tribune Tower's embedded stones, the Rookery's glass atrium, the Sullivan Center's cast iron: these are not detours from the city — they are the city.

Several spots deliberately span multiple tags: the Green Mill is music, jazz, and nightlife. The Chicago Theatre is architecture and music. This cross-tagging means a single-tag query still surfaces multi-dimensional places, and a multi-tag query creates natural density in the resulting route.

03 — Design

Dark, moody, specific — the aesthetic decisions

The color system is anchored in the Chicago municipal flag: two horizontal cyan stripes and four red stars on white. For a dark interface, this translates to deep navy backgrounds with cyan as the primary interactive color and red used sparingly for emphasis. The result reads as civic without being literal.

Each tag gets its own color that carries consistently across filter buttons, card left borders, map markers, and tag pills — creating a visual system where color alone conveys category. The tag filter buttons use a physical toggle metaphor: inset shadow at rest (pressed down), lift and glow on hover, filled and lit when active.

04 — Logic

The tag system and nearest-neighbor routing

Filtering is additive: selecting multiple tags returns spots matching any of them, not all. This maximizes the usefulness of combined selections — "jazz + café" returns a richer, more varied day than a strict AND would.

The route algorithm is a nearest-neighbor heuristic starting from Millennium Park (the geographic and cultural center of the tourist zone). For each step, it selects the unvisited spot closest to the current position by haversine distance — the spherical-earth calculation that correctly accounts for longitude distortion at Chicago's latitude (~42°N).

1
Millennium Park (anchor)
2
Nearest unvisited spot
3
Next nearest from #2…
N
All filtered spots covered

The result isn't mathematically optimal (the Traveling Salesman Problem is NP-hard), but it's geographically sensible — producing routes that cluster nearby spots rather than zigzagging across the city. For 25 spots or fewer, nearest-neighbor typically gets within 15–20% of optimal without any perceptible lag.

Cards and map markers are bidirectionally synced: clicking a card pans the map and opens its popup; clicking a marker scrolls the corresponding card into view and highlights it. The route sequence number appears on both.

05 — The Map

Dark tiles with no API key — a CSS trick

Leaflet.js with OpenStreetMap tiles is entirely free and requires no API key. The challenge was making the default light-colored OpenStreetMap tiles work with a dark interface — without switching to a paid dark tile provider.

The solution is a single CSS filter applied to Leaflet's tile layer container. The combination inverts the tile image, hue-rotates it to undo the color inversion, and desaturates it slightly to produce a dark, moody base map that doesn't compete with the colored markers:

.leaflet-tile-pane {
  filter: brightness(0.55) saturate(0.35) invert(1) hue-rotate(180deg); /* dark map, no API key */
}

Route markers use Leaflet's L.divIcon — custom HTML injected as map markers — so they inherit the same tag color system as the sidebar cards. The route itself is drawn as a dashed cyan polyline that updates live on every filter change.
